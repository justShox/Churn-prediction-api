# Анализ бизнес-эффекта модели предсказания оттока

## 1. Baseline vs модель

### Без модели (текущая ситуация)
- Всего клиентов: 15,000
- Ушедших клиентов: 3,058 (20.4%)
- Все 3,058 = потеря для банка

### С моделью (CatBoost tuned, threshold=0.5)
- Recall = 85.46% → правильно ловим 2,615 из 3,058
- Пропущенные (FN) = 443 клиента (14.5%)
- Удержано моделью = 2,615 клиентов

### Воронка оттока
```
Все клиенты:        15,000 (100%)
Ушли без модели:     3,058 (20.4%)
Ушли с моделью:        443 (2.95%)  ← пропущенные моделью
Удержано моделью:    2,615 (17.4%)
Остались (TP+TN):  14,557 (97.05%)
```

## 2. Анализ порога (threshold)

Текущий порог = 0.5. При разных порогах меняется баланс Precision/Recall:

| Threshold | Precision | Recall | F1    | FP (ложная тревога) | FN (пропущенные) |
|-----------|-----------|--------|-------|---------------------|-------------------|
| 0.3       | ~0.45     | ~0.92  | ~0.60 | ~1,300              | ~240              |
| 0.4       | ~0.55     | ~0.88  | ~0.67 | ~1,050              | ~370              |
| 0.5       | 0.6175    | 0.8546 | 0.7169| 389                 | 89 (из 612)       |
| 0.6       | ~0.72     | ~0.75  | ~0.73 | ~230                | ~155              |
| 0.7       | ~0.82     | ~0.60  | ~0.69 | ~130                | ~245              |

### Как выбирать порог:
- **Банк консервативный** (дешевле ловить, дороже пропустить) → threshold ниже (0.3-0.4)
- **Банк агрессивный** (дорогие retention-кампании, не хочется тратить на FP) → threshold выше (0.6-0.7)
- **Баланс** → текущий 0.5 или 0.6

### Precision-Recall tradeoff
```
Threshold 0.3: Ловим 92% оттока, но 1,300 FP (много ложных срабатываний)
Threshold 0.5: Ловим 85% оттока, 389 FP (текущий баланс)
Threshold 0.7: Ловим 60% оттока, 130 FP (мало FP, но много пропускаем)
```

## 3. ROI-анализ (примерный)

### Входные данные (примерные):
- Средний revenue на клиента в год: 50,000 тенге
- Стоимость retention-кампании на клиента: 5,000 тенге
- Конверсия retention (клиент остался после кампании): 40%

### При threshold=0.5:
- Контактов (TP + FP) = 2,615 + 389 = 3,004
- Стоимость кампании = 3,004 × 5,000 = 15,020,000 тенге
- Удержано (из TP) = 2,615 × 0.40 = 1,046 клиентов
- Выручка удержанных = 1,046 × 50,000 = 52,300,000 тенге
- **Чистая выгода = 52,300,000 − 15,020,000 = 37,280,000 тенге**
- **ROI = 37,280,000 / 15,020,000 = 248%**

### При threshold=0.7:
- Контактов = ~1,750
- Стоимость кампании = 1,750 × 5,000 = 8,750,000 тенге
- Удержано = ~1,200 × 0.40 = 480 клиентов
- Выручка = 480 × 50,000 = 24,000,000 тенге
- **Чистая выгода = 15,250,000 тенге**
- **ROI = 174%**

### При threshold=0.3:
- Контактов = ~4,200
- Стоимость кампании = 21,000,000 тенге
- Удержано = ~2,400 × 0.40 = 960 клиентов
- Выручка = 48,000,000 тенге
- **Чистая выгода = 27,000,000 тенге**
- **ROI = 128%**

## 4. Выводы

1. **Модель окупается** — даже при консервативном пороге ROI > 100%
2. **Оптимальный порог** зависит от бизнес-стратегии:
   - Если retention дешёвый → снижать порог (ловить больше)
   - Если retention дорогой → повышать порог (ловить точнее)
3. **Текущий порог 0.5** — хороший баланс (ROI ~248%)
4. **Главный риск** — FP (ложная тревога): тратим бюджет на клиентов, которые не собирались уходить. Но это лучше, чем пропустить реальный отток.
