PROJECT SUMMARY — Bank Customer Churn Prediction
==================================================

1. DATA
-------
Dataset: data/raw/TZ.csv
Rows: 15,000
Features (raw): 14 columns (ID, ID_клиента, фамилия + 11 признаков + таргет)
Features after cleaning: 11 input + 1 target (ушел_из_банка)
Churn rate: 20.4% (3,058 churned / 15,000 total)
Class ratio: 1:3.9 (retained:churned)

Split: 80/20, stratified, random_state=42
Train: 12,000 | Test: 3,000


2. FEATURE ENGINEERING (02_preprocessing.ipynb)
----------------------------------------------
Base features (11):
  1. кредитный_рейтинг → credit_score (float, 300–850)
  2. город → city (categorical: Алматы, Астана, Атырау)
  3. пол → gender (categorical: Male, Female)
  4. возраст → age (float, 18–100)
  5. стаж_в_банке → years_with_bank (float, 0–50)
  6. баланс_депозита → deposit_balance (float, 64.6% NaN → filled with 0)
  7. число_продуктов → num_products (float, 1–4)
  8. есть_кредитка → has_credit_card (binary: 0/1)
  9. активный_клиент → is_active_member (binary: 0/1)
  10. оценочная_зарплата → estimated_salary (float)
  11. есть_депозит → has_deposit (binary, derived from баланс_депозита notna)

OHE (3 features):
  12. city_Астана
  13. city_Атырау
  14. gender_Male

Feature Engineering (6 features):
  15. product_risk_level — маппинг num_products: {1→2, 2→0, 3→3, 4→3}
  16. balance_to_salary_ratio — deposit_balance / (estimated_salary + 1)
  17. active_single_product — is_active==1 & num_products==1
  18. loyal_high_balance — years_with_bank>=5 & deposit_balance>50000
  19. young_no_card — age<30 & has_credit_card==0
  20. age_tenure_interaction — age * years_with_bank
  21. salary_products_interaction — estimated_salary * num_products

Dropped (multicollinearity): age_squared, age_group, credit_score_squared

Final features: 19 (after dropping 3 multicollinear)

7. API
------
Endpoints:
  GET  /              — service status check
  GET  /health        — health check
  POST /predict       — single customer prediction
  POST /predict_batch — batch prediction (list of customers)

Input features (CustomerData — русские названия):
  кредитный_рейтинг:  float (300–850)
  город:              str (Алматы | Астана | Атырау)
  пол:                str (Male | Female)
  возраст:            float (18–100)
  стаж_в_банке:       float (0–50)
  баланс_депозита:    float (≥0)
  число_продуктов:    float (1–4)
  есть_кредитка:      float (0 or 1)
  активный_клиент:    float (0 or 1)
  оценочная_зарплата: float (≥0)
  есть_депозит:       float (0 or 1)

Output (PredictionResponse):
  churn_probability:  float  — probability of churn (0–1)
  prediction:         int    — binary (1=churn, 0=stay)
  risk_level:         str    — low (<0.4), medium (0.4–0.7), high (>0.7)

Preprocessing in API (app.py pipeline):
  1. OHE: город → city_Астана, city_Атырау; пол → gender_Male
  2. Feature Engineering: 9 признаков (product_risk_level, age_group, age_squared, ...)
  3. Rename Rus → Eng (column_mapping из preprocessing.pkl)
  4. Scaler (StandardScaler из preprocessing.pkl)
  5. Reorder к final_features (автоматически удаляет age_squared, age_group)

Threshold: 0.5 (default), переключается через USE_CALIBRATED в app.py
